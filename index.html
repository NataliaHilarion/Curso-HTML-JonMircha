Introducción a HTML

HTML (HyperText Markup Language) es el lenguaje estándar para crear y estructurar páginas web. Utiliza una serie de elementos y etiquetas que describen el contenido y la estructura de una página, como encabezados, párrafos, enlaces, imágenes, y formularios.

Etiquetas de Bloque vs Etiquetas de Linea

En HTML, las etiquetas de bloque, como div, p y header, estructuran el contenido creando secciones que ocupan todo el ancho disponible y siempre comienzan en una nueva línea, organizando el layout de la página. Por otro lado, las etiquetas en línea, como span, a y em, se integran dentro del flujo del texto y sólo ocupan el espacio necesario, permitiendo estilizar partes específicas del contenido sin interrumpir el flujo

La etiqueta de bloque por excelencia es la div, una etiqueta de bloque es aquella que genera saltos de linea con sus etiquetas hermanas
La etiqueta de linea por excelencia es la span, una etiqueta de linea solo ocupa el espacio necesario que tiene su contenido Hola soy otra span

Imagenes

La etiqueta img en HTML se utiliza para incorporar imágenes en una página web. Es un elemento vacío, lo que significa que no tiene una etiqueta de cierre y se define mediante atributos como src source para especificar la URL de la imagen, y alt (alternate text) para proporcionar una descripción alternativa de la imagen. La etiqueta Title para que agregue nombre al pasar el cursor sobre la imagen.

SVG - vectores

SVG Scalable Vector Graphics es un formato de imagen basado en XML que se utiliza para definir gráficos vectoriales bidimensionales en HTML. A diferencia de las imágenes rasterizadas, como JPEG o PNG, SVG puede escalarse a cualquier tamaño sin pérdida de calidad, lo que lo hace ideal para gráficos en la web que necesitan ser responsivos. Además, SVG permite la manipulación a través de CSS y JavaScript, facilitando la creación de gráficos interactivos y animaciones. Al ser un estándar abierto, es ampliamente compatible con la mayoría de los navegadores modernos.

Listas

Las listas en HTML permiten organizar información en forma de ítems secuenciales o desordenados. Existen tres tipos principales de listas: las listas no ordenadas ul - Unorder list, que presentan los ítems con viñetas; las listas ordenadas ol - Order List, que enumeran los ítems con números; y las listas de definición dl - Definition List, que se usan para términos y sus descripciones. Los elementos individuales de una lista se definen con la etiqueta li en el caso de listas ordenadas y no ordenadas, y con dt (definition term) y dd (description data) en el caso de listas de definición. Estas estructuras son fundamentales para mejorar la organización y legibilidad del contenido en una página web.

Enlaces Internos

Los enlaces internos en HTML te permiten navegar a diferentes secciones dentro de la misma página. Para crear enlaces internos, utilizas el atributo id en el elemento de destino y el atributo href en el enlace que apunta a ese destino.

Etiquetas de Audio y Video

Las etiquetas Audio y Video en HTML se utilizan para incrustar archivos de audio y video directamente en una página web sin necesidad de plugins externos

Atributos Principales:

  1. Src: Especifica la URL del archivo de audio que se va a reproducir. Este es el atributo principal si no se utilizan elementos anidados.
  2. Controls: Agrega controles predeterminados para el audio, como reproducir, pausar y ajustar el volumen, permitiendo la interacción del usuario.
  3. Autoplay: Hace que el audio se reproduzca automáticamente cuando la página se carga, aunque su uso debe ser cuidadoso, ya que puede resultar molesto para los usuarios.
  4. Loop: Hace que el audio se repita en bucle continuo una vez que termina, sin necesidad de intervención manual.
  5. Preload: Indica al navegador cómo manejar la precarga del archivo de audio. Puede tomar valores como auto (precarga automática), metadata (solo metadatos), o none (no precargar), afectando el rendimiento y el tiempo de carga.

Iframe

No se recomienda usar la etiqueta iframe en HTML porque puede presentar problemas de seguridad, como vulnerabilidades de clickjacking, y afectar negativamente el rendimiento del sitio web al aumentar el tiempo de carga. Además, los iframe pueden dificultar la accesibilidad y el SEO, ya que el contenido dentro del iframe no es fácilmente indexable por los motores de búsqueda, y puede no ser completamente accesible para tecnologías de asistencia, como los lectores de pantalla

Se puede usar tambien para agregar pdf, google maps o videos en YouTube. Pero No es recomendable tener mas de un Iframe en HTML
